# v0.2 cost-attribution redesign — full design (opus design agent, 2026-07-08)

Anchor of v0.2. Verified against src/costs.ts. The flagship differentiator
(per-seat cost) is broken in the normal multi-seat-same-repo case; this fixes it.

## Diagnosis confirmed against the code

- **#1 subagent leak** — `sessionFiles()` (costs.ts:27-61) yields `[absPath, proj]`.
  For a subagent file it discards the fact that fixes everything: the parent
  session directory name is `entry` at costs.ts:49 (`join(projDir, entry,
  "subagents")`), and for flat logs `<session>.jsonl` the session id equals that
  name. `computeCosts` tallies the subagent record under its OWN `d.sessionId`
  (costs.ts:114), never in `seatSessions` → `seat_id: null` (costs.ts:146). Parent
  sits right there, unused.
- **#2 same-cwd collapse** — both `findSessionIdByHeuristic` (costs.ts:164,
  `fresh.length !== 1 → null`) and `attributeSeatsToSessions` (costs.ts:227,
  `hits.length === 1` + cross-seat-unique at :260) are mtime/first-timestamp
  window matchers. Two seats booting in one cwd inside 120s → ≥2 candidates →
  both null. The standard fleet-in-one-repo launch is exactly this.
- **#3 O(all history) on sync path** — `handleCosts` (broker.ts:228) →
  `computeCosts` readFileSync's every jsonl under ~/.claude/projects
  (costs.ts:89-92) on every `/costs`; `patrol status` (status.ts:9-12) fires
  `/costs` on the interactive path.
- **#4 history dies with the seat** — attribution lives only in the live `seats`
  row; `handleUnregister` (broker.ts:216) deletes it; `BROKER_START`
  (broker.ts:35) resets the default window on restart. No durable ledger.
- **#5 restart misattribution** — mtime/first-ts matching can bind a restarted
  seat to its PREVIOUS log.

## Core decision: exact attribution = launcher-issued token, resolved by content match

The one signal fully under our control is what the launcher writes: `SeatSpec.prompt`
+ env (compose.ts:88-111). A globally-unique per-seat token injected into BOTH env
and launch prompt is content-matched (not mtime) → immune to N-seats-same-cwd (#2),
and self-locates inside the transcript (broker joins token→session with zero pid games).

**SPIKE PASSED (2026-07-08):** `claude -p "... [patrol-seat: cp-0375a012]"` → marker
present in the session jsonl `user` record; session id = jsonl filename stem.
Refinement: marker also appears in last-prompt/queue-operation records → the broker
token→session scan should be a plain substring grep over the file, not tied to a
record type.

### Attribution layers, precedence high→low

**Layer 1 — prompt-marker token (exact; launcher seats). PRIMARY.**
1. `up.ts` generates `token = "cp-" + 8hex` per seat, passes into `composeSeat`.
2. `composeSeat` adds `CLAUDE_PATROL_SEAT_TOKEN=<token>` to env (compose.ts:104) and
   appends to the prompt argv: if `spec.prompt` set, `spec.prompt + "\n\n[patrol-seat:
   <token>]"` (compose.ts:102); if unset, minimal `"[patrol-seat: <token>] You are
   seat <role>. Await instructions."`.
3. `seat-server.ts main()` reads env, sends `seat_token` in `/register` (seat-server.ts:252-262).
4. Broker resolves `seat_token → session_id` by scanning the seat's project dir for
   the one jsonl containing the token; caches binding permanently in `seat_runs`.
   File may not exist at register time (documented race) — background indexer retries
   until it lands.

**Layer 2 — SessionStart hook (exact; any seat incl. manual; VERIFY then adopt).**
CC hooks receive `session_id` + `transcript_path` on stdin. New plugin SessionStart
hook (sibling of plugin/hooks/dereg.ts) POSTs `{session_id, transcript_path, cwd,
ppid}` to a new `/observe-session` route; broker binds it to a seat_run. Upgrades
MANUAL seats to exact.
- **Kill criterion:** if this CC build doesn't pass session_id/transcript_path to
  SessionStart hooks, OR the `$PPID` can't join a seat row (note: seat-server.ts:253
  registers `process.pid` = MCP-server pid, while dereg hook uses `process.ppid` =
  claude pid — join-key mismatch to resolve or sidestep via cwd+time). If either
  fails, drop Layer 2; Layers 1+3 stand. This is why Layer 1 is primary: one
  controllable mechanism, zero unverified assumptions.

**Layer 3 — window heuristic (fallback; unattributed-safe).** Keep
`attributeSeatsToSessions`/`findSessionIdByHeuristic` verbatim for seats with neither
token nor observation. Exactly-one-or-null, cross-seat unique → a manual seat in a
busy cwd degrades to unattributed, never wrong.

### Alternatives killed
- Read a CC-exposed session id in the seat-server — `discoverSessionId`
  (seat-server.ts:102) already assumes CC hands nothing to spawned MCP servers; only
  `CLAUDE_PATROL_SESSION_ID` override exists. Killed as primary; kept as override
  fast-path.
- fs-watch for the new jsonl at register time — can't disambiguate two files appearing
  together in one cwd; reintroduces #2. Killed.
- Broker-issued correlation id in a sidecar — same pid-join fragility as Layer 2
  without the token's content-match certainty. Killed.

Token trade-off: Sacrificed: a prompt-less seat takes one cheap boot turn to carry the
marker; briefing templates must tell the model to ignore `[patrol-seat: …]` lines.
Acceptable: fleet seats almost always ship a briefing prompt (DESIGN D4.4), marker
appends free; the dark case is exactly launcher fleets, which this fixes. Reverses if:
users need truly silent idle seats → add a `silent` seat flag skipping marker
injection, Layer 3 for those seats only.

## (b) Subagent → parent-seat rollup
1. `sessionFiles` — yield third element: `[absPath, proj, parentSessionId]`, where
   `parentSessionId = entry` (the `<session>` dir) for subagent files, `null` for flat.
2. `computeCosts` — per record compute `attrSessionId = parentSessionId ?? d.sessionId`.
   Keep displayed `session_id` as the record's own id (subagent visibility), but set
   `seat_id = seatSessions.get(attrSessionId)` (costs.ts:146) and project-scoping
   `attributed = seatSessions.has(attrSessionId)` (costs.ts:122). Store `attrSessionId`
   on Tally. Dedupe stays `(ownSessionId, msgId)` — subagent msg ids distinct.
   Result: executor/subagent spend rolls up under parent seat, model split preserved.
   Reverses if the board wants a subagent breakdown → additive `parent_session_id?`
   on CostRow (escalation).

## (c) Persisted seat-run history (SQLite) — additive CREATE TABLE IF NOT EXISTS

```
seat_runs(
  run_id INTEGER PRIMARY KEY AUTOINCREMENT,
  seat_id TEXT NOT NULL, session_id TEXT, seat_token TEXT,
  cwd TEXT NOT NULL, role TEXT, model TEXT, profile TEXT,
  registered_at TEXT NOT NULL, ended_at TEXT     -- null = live
)  -- index (session_id), (seat_token), (ended_at)

cost_ledger(
  session_id TEXT NOT NULL, attr_session_id TEXT NOT NULL, model TEXT NOT NULL,
  bucket_ts INTEGER NOT NULL,                      -- ts floored to the hour
  input INTEGER, output INTEGER, cache_write INTEGER, cache_read INTEGER,
  PRIMARY KEY (session_id, model, bucket_ts)
)

session_index(file_path TEXT PRIMARY KEY, parent_session_id TEXT,
  bytes_parsed INTEGER NOT NULL, mtime_ms INTEGER NOT NULL)

seen_msgs(session_id TEXT, msg_id TEXT, PRIMARY KEY(session_id,msg_id))
```

- `handleRegister` (broker.ts:145): keep live `seats` row for messaging; also INSERT a
  `seat_runs` row. Keep the existing session_id uniqueness guard as env-override fast path.
- `handleUnregister` (broker.ts:216): `UPDATE seat_runs SET ended_at=? WHERE seat_id=?
  AND ended_at IS NULL` — mapping survives. Still delete the live `seats` row.
- `/costs` joins `cost_ledger.attr_session_id → seat_runs.session_id` across all runs
  overlapping the window → killed/unregistered seat history still attributes (#4). A
  restart gets a fresh seat_runs row; previous run keeps its own → restart can't steal
  the previous log (#5, exact-by-token not mtime).
  DB growth prunable via the 7-day purge pattern (broker.ts:102).

## (d) Move /costs off the synchronous status path
1. Background `indexTick()` on setInterval (mirror cleanStaleSeats broker.ts:106-107,
   ~10-15s) + one cold call at startup. Each tick: prefilter to project dirs of interest
   (live seats' projectDirName(cwd) + dirs already in session_index), stat each candidate,
   skip when (size,mtime) unchanged. For changed files, resume from bytes_parsed, parse
   only the appended tail. If size < bytes_parsed (rewrite/truncate) full reparse + reset
   that file's ledger contribution. seen_msgs guards resume-rewrite dup. Upsert per
   (session,model,hour) deltas carrying attr_session_id. Resolve unresolved seat_runs:
   Layer-1 token scan else Layer-3 heuristic; write back session_id when unambiguous.
2. `handleCosts` becomes table reads only: sum cost_ledger within [since,until], left-join
   seat_runs for seat_id, price via priceFor in JS over grouped rows. No FS walk on the
   request path. since still defaults to BROKER_START but the ledger predates restarts.
3. Keep `computeCosts` as the pure reference impl (unit tests + single-file ledger
   contribution — factor the assistant-usage line parse into a shared helper so the
   incremental indexer and computeCosts don't diverge).
   Sacrificed: /costs up to one tick stale; first cold index still O(all history) once.
   Acceptable: broker is long-lived; status needn't be sub-second fresh. Reverses if
   cold-start latency matters → bound the cold pass to a lookback window, lazy-backfill.
